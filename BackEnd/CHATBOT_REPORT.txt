🚀 DEPLOYMENT INSTRUCTIONS
════════════════════════════════════════════════════════════════════

1. Environment Setup:
   Create .env file in BackEnd/ folder:
   
   DB_HOST=your_postgres_host
   DB_PORT=5432
   DB_USER=postgres_user
   DB_PASSWORD=secure_password
   DB_NAME=pharmacy_db
   PORT=5000
   CORS_ORIGIN=http://localhost:5173

2. Install Dependencies:
   cd BackEnd
   npm install

3. Database Setup:
   npx psql -U postgres -c "CREATE DATABASE pharmacy_db"
   node seed-medicines.js

4. Start Server:
   node src/server.js
   # or for development with auto-reload:
   nodemon src/server.js

5. Backend will be available at:
   http://localhost:5000
   
6. Frontend API endpoints:
   Chat: POST http://localhost:5000/api/chat/chat
   Agent: POST http://localhost:5000/api/agent/chat

🔧 TROUBLESHOOTING
════════════════════════════════════════════════════════════════════

Issue: Server won't start
Solution:
  1. Check Node.js version (16+ required)
  2. Verify .env file exists with DB credentials
  3. Check if port 5000 is available
  4. Review error logs

Issue: Database connection fails
Solution:
  1. Verify PostgreSQL is running
  2. Check DB_HOST, DB_USER, DB_PASSWORD in .env
  3. Ensure database exists
  4. Run: npx psql -U postgres -c "CREATE DATABASE pharmacy_db"

Issue: Chatbot returns "Medical Pharmacy Assistant" message
Solution:
  1. Query might be non-medical (expected behavior)
  2. If medical query, check if keywords are recognized
  3. Review medical keywords list above

Issue: No medicines found in database
Solution:
  1. Run seed script: node seed-medicines.js
  2. Verify medicines table exists
  3. Check database connection
